Claves de cruce (conciliación)

  • order_id (ML) ↔ payment_id (MP) vía external_reference
  • pack_id (ML) agrupa múltiples órdenes en un envío
  • source_id (MP) se vincula con payment_id o external_reference

3. Login Gerencia

Reutiliza el modelo Usuario existente, que ya tiene nivelAcceso: Int y FK a Rol.

RequisitoValidación
Usuario existedb.usuario.findUnique({ where: { usuario } })
Nivel correctouser.nivelAcceso === 0
Rol gerenciauser.rol.nombre === 'gerencia'
Activouser.activo === true
Passwordbcrypt.compare(password, user.password)
SesiónJWT + cookie httpOnly (24 h)
  • Frontend Tercer tab "Gerencia" en login-page.tsx
  • Backend API POST /api/auth/login-gerencia
  • Backend Extender SessionUser y type Rol en auth.ts

4. Menú y navegación

En src/app/page.tsx:

  • Import dinámico de GerenciaTab
  • TabsTrigger visible solo si user.nivelAcceso === 0
  • TabsContent renderiza el componente
  • Protección en backend: todos los endpoints usan requireRole('gerencia')

5.4 Conciliación

Matriz de cruce orden ↔ pago ↔ movimiento. Alertas:

  • ORDEN_SIN_PAGO Orden sin pago asociado
  • PAGO_SIN_ORDEN Pago sin orden correspondiente
  • DIFERENCIA_IMPORTE Monto orden ≠ monto pago
  • DINERO_RETENIDO Pago liberado parcialmente
  • REEMBOLSO_NO_CONCILIADO Reembolso sin orden de origen
  • CONTROCARGO_PENDIENTE Disputa activa

5.5 Reportes Gerenciales

  • Filtros: fecha, producto, estado, canal
  • Exportar Excel (.xlsx)
  • Exportar CSV
  • Resumen diario / semanal / mensual

Crear 10 modelos Prisma, seed de rol + usuario gerencia, prisma db push.

API login-gerencia, extender auth.ts, tercer tab en login-page.tsx.

9. Seguridad

  • Seguridad Todos los endpoints de gerencia validan requireRole('gerencia') + nivelAcceso === 0
  • Seguridad Tokens ML/MP almacenados cifrados en SellerAccount
  • Seguridad Sin exponer tokens en frontend ni en logs
  • Seguridad Variables de entorno para todos los client_id/client_secret
  • Seguridad Auditoria de cada sincronización en AuditoriaSincronizacion
  • Seguridad Control de HTTP 206 (datos parciales), 429 (rate limit), 401/403 (tokens)

10. Riesgos y controles
