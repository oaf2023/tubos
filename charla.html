¡Bug raíz encontrado!

En src/app/page.tsx la condición era: 
user.rol === 'gerencia'   // ✖ SIEMPRE false
El API /api/auth/login-gerencia devuelve rol como objeto Prisma { id: 1, nombre: 'gerencia' }, no como string. La comparación estricta con un string 'gerencia' siempre fallaba, ocultando la pestaña Gerencia para todos los usuarios desde que se introdujo la restricción.

Fix: 
user.rol?.nombre === 'gerencia'   // ✓ correcto
